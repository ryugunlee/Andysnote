Before implementing any Markdown features, I want to redesign the overall architecture of AndysNote.

Do NOT write or modify any code yet.

This is an architecture planning phase.

I want AndysNote to become a long-term project that is easy to maintain and extend.

Instead of focusing only on Markdown, I want to establish a clean layered architecture for the entire application.

---

## Goal

Please review the current codebase and propose a modular architecture that can support future growth without major rewrites.

The architecture should prioritize:

- maintainability
- modularity
- scalability
- low coupling
- high cohesion
- independent components

---

## Core design principles

Each module should have only one responsibility.

Modules should communicate through clear interfaces.

Avoid putting unrelated logic into the same file.

Future features should require adding new modules instead of modifying existing ones whenever possible.
